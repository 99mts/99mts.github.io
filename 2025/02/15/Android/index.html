活动状态： 运行 暂停（可见） 停止（不可见）销毁

活动的生存期：

onCreate() 当活动第一次被创建时调用

onStart() 当活动由不可见变为可见时调用

onResume() 当活动进入前台并可以与用户交互时调用

onPause() 当活动失去焦点但仍部分可见时调用（例如弹出对话框或切换到其他Activity）

onStop() 当活动完全不可见时调用

onDestroy() 当活动被销毁时调用

onRestart() 当活动从停止状态重新启动时调用（即从不可见到可见）

常见场景分析

  1. 启动Activity

    onCreate() → onStart() → onResume()

  2. 切换到另一个Activity

    当前Activity：onPause() → onStop()

    新Activity：onCreate() → onStart() → onResume()

  3. 返回到原Activity

    原Activity：onRestart() → onStart() → onResume()

  4. 退出Activity

    onPause() → onStop() →onDestroy()

活动的启动模式 :

1. standard（标准模式）每次启动都创建新实例，压入栈顶

2. singleTop（栈顶复用模式）如果目标Activity已在栈顶，则复用；否则创建新实例

3. singleTask（栈内复用模式）如果目标Activity已存在，则复用并清空其上的Activity；否则新建

4.singleInstance（单实例模式） 独立任务栈，全局唯一实例
