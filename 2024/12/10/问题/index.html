搭建个人博客时遇到的问题

v 2.3

0.教程推荐

如何用Hexo搭建个人博客?

视频版教程：bilibili视频链接

1.少用steam++（ Watt Toolkit ）

​ 虽然用steam++有时能很快进入GitHub，但是使用steam++进行加速时，sream++会更改host文件（好像是这样吧），然后引发一些隐性问题，比如导致在使用hexo执行命令时报错。如果遇到GitHub因为网络问题打不开，可以在网址前面加上kk（就像这样kkgithub）使用镜像网站，可以解决大概90%的网络问题。

2.not found git

​ 忘了是在哪一步（好像是用hexo g生成的时候），在输入命令之后没反应，返回了一个 not found git 。但是我记得我下了Git，然后就卡住了。最后发现是之前有一个命令输入错了，只需要回到博客文件夹打开git bash输入这个代码npm install hexo-deployer-git –save（安装自动部署发布工具）就可以了。

3.hexo d部署博客问题 （特别鸣谢 yxl）

​ 在使用hexo d部署博客时大概率会遇到这个问题，就是使用hexo d命令时出现这些报错

Error: Spawn failed
at ChildProcess. (D:\StudyMaterials\8_FileManagement\testBlog\blog\node_modules\hexo-util\lib\spawn.js:51:21)
atChildProcess.emit(events.js:314:20)atChildProcess.cp.emit(D:\StudyMaterials\8_FileManagement\testBlog\blog\node_modules\crossspawn\lib\enoent.js:34:29)atProcess.ChildProcess._handle.onexit (internal/child_process.js:275:12)

此时可以打开config.yml配置文件,

将repo: https://github.com/yourname/yourname.github.io.git

改为repo: git@github.com:yourname/yourname.github.io.git

即把网址中的

https://github.com/

改为

git@github.com:

就可以了（注意不要漏了 斜杠 和 冒号 ）

4.其他问题

1） ： （md文档中 ）冒号后面要有一个空格。

2） 如果使用hexo init命令时，返回一个bash: hexo: command not found。只需将命令hexo init改为npx hexo init即可。

3） 在使用hexo时不要用快捷键，Ctrl+C在 hexo里面的意思是停止运行，不是复制。

4）打开md文档之后最好不要动那个tags：，因为我动了之后出现了奇怪的问题。
