  • 抽象

    抽象类不能用 new 关键字创建对象

    抽象类主要用于作为父类定义一个模板，让子类继承并实现其方法。

    应用场景 ：

    当父类知道每个子类都要执行某个行为，但具体实现方式不同。

    父类可以定义抽象方法，交给子类去重写实现。

    好处 ：

    通过抽象类和抽象方法，可以更好地支持面向对象编程中的多态特性

    抽象类可以包含一些通用的方法实现供子类继承使用，有利于代码复用

    抽象类可以强制子类实现某些方法，确保子类的行为符合预期

  • 接口

    Java 提供了一个关键字 interface 来定义接口

    实现类必须重写接口中所有的抽象方法，否则该类需要被定义为抽象类

    一个类可以同时实现多个接口，接口不能创建对象

    // 定义接口 InterfaceA
    public interface InterfaceA {void methodA();}

    // 定义接口 InterfaceB
    public interface InterfaceB {void methodB();}

    // 实现类 M 实现了两个接口
    public class M implements InterfaceA, InterfaceB {
    @Override
    public void methodA() {
    System.out.println(“Method A implemented”);
    ​ }

    @Override
    public void methodB() {
    System.out.println(“Method B implemented”);
    ​ }

    }
